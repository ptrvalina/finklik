# Бэклог спринта 18 — отложенные задачи (не забыть)

Сюда перенесены пункты, которые **требуют внешних API госорганов**, **ЭЦП**, **официальных схем**, **инфраструктуры** или явно не делались в спринтах 11–12. Источник: [`PRODUCT_SPRINT_ALIGNMENT.md`](PRODUCT_SPRINT_ALIGNMENT.md), [`scaling-plan.md`](../pilot/scaling-plan.md), обсуждения разработки.

## Снято с бэклога (уже в продукте)

- **UI:** в «Настройки → Подача отчётов → Просмотр» для **accepted/rejected** с флагом `has_submission_snapshot` подгружается архив (`GET /submissions/{id}?include_snapshot=true`), отдельный блок предпросмотра и экспорт файлов по данным архива.

## Из продуктовых спринтов 11–12 (отчётность и порталы)

| Задача | Почему в спринте 18 |
|--------|---------------------|
| Официальные **XML / XSD** и прочие форматы порталов (ПУ-3, УСН, НДС и др.) | Нужны актуальные схемы и согласование с органами |
| **Валидация** черновика по схемам портала | Зависит от форматов выше |
| **Редактирование** черновика отчёта перед подачей (полноценное) | Продуктовый объём + UX; частично закрыто предпросмотром |
| **Версионирование форм** при смене нормативки | Реестр версий + миграции данных |
| Реальные **API ФСЗН / МНС / Белгосстрах / Белстат** | Договоры, контуды, тестовые стенды |
| **ЭЦП** и криптопровайдер | Аппаратные/сертифицированные решения |
| Промежуточный статус **`submitted`** до ответа портала (если нужен отдельно от текущего потока) | Обычно очередь/async + реальный обмен |
| **Очередь** (Celery/RQ) для долгой подачи и ретраев вместо синхронного HTTP | Инфраструктура фазы 2–3 |
| **Архив** квитанций/подписей в **S3** и долгий retention | Инфра + комплаенс |
| **WebSocket на фронте** для `report_status` (тосты без перезагрузки; сейчас только блок в просмотре по запросу) | UX |

## Уже без внешних API (сделано в коде, не дублировать в бэклоге)

- Мок / `MOCK_SUBMISSION_REJECT_RATE`, `portal_sim` (DEBUG), режим **`SUBMISSION_PORTAL_MODE=http`** к **своему** адаптеру
- Уведомления: **WebSocket** + **email** после ответа «портала»
- **`submission_snapshot_json`**: снимок данных на момент подачи + `GET /submissions/{id}` для просмотра архива
