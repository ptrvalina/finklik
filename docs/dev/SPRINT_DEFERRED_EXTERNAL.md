# Спринт «отложено до внешних сервисов»

Здесь собраны задачи из продуктовой дорожной карты и смежных обсуждений, которые **нельзя завершить в коде без реальных внешних систем**: API госорганов, договоры с банками, ЭЦП, официальные XML/XSD, платёжные шлюзы, managed-инфраструктура под прод и т.п.

**Критерий:** если для «готово» нужен контракт, тестовый контур органа, сертифицированный криптопровайдер или оплаченный облачный сервис — пункт лежит здесь, а не в [`TECH_CATCHUP_ORDERED.md`](TECH_CATCHUP_ORDERED.md).

Связанные документы: [`../pilot/scaling-plan.md`](../pilot/scaling-plan.md), ранее [`BACKLOG_SPRINT18_DEFERRED.md`](BACKLOG_SPRINT18_DEFERRED.md) (см. указатель внизу).

---

## Регламентированная отчётность и госпорталы (продуктовые спринты 11–12)

| Задача | Почему вне текущего кода |
|--------|---------------------------|
| Официальные **XML / XSD** и прочие форматы порталов (ПУ-3, УСН, НДС и др.) | Нужны актуальные схемы и согласование с органами |
| **Валидация** черновика по схемам портала | Зависит от форматов выше |
| **Полноценное редактирование** черновика отчёта перед подачей | Большой продуктовый объём; частично закрыто предпросмотром и архивом |
| **Версионирование форм** при смене нормативки | Реестр версий + миграции данных |
| Реальные **API ФСЗН / МНС / Белгосстрах / Белстат** | Договоры, контуды, тестовые стенды |
| **ЭЦП** и интеграция с криптопровайдером | Аппаратные/сертифицированные решения |
| Промежуточный статус **`submitted`** до ответа портала (если отдельно от текущего потока) | Обычно очередь + реальный обмен |
| **Очередь** (Celery/RQ) для долгой подачи и ретраев вместо синхронного HTTP | Инфраструктура фазы 2–3 |
| **S3** (или аналог) для квитанций/подписей и долгий retention | Инфра + комплаенс |
| **WebSocket на фронте** для `report_status` (тосты без перезагрузки) | UX; бэкенд-события уже есть |

**Уже сделано без внешних API** (не дублировать как «хвост»): мок/`http`-адаптер подачи, WS+email, архивный снимок `submission_snapshot_json`, UI подачи и просмотра — см. [`PRODUCT_SPRINT_ALIGNMENT.md`](PRODUCT_SPRINT_ALIGNMENT.md).

## Инфраструктура и масштаб (чекбоксы фаз 1–3 в `scaling-plan.md`)

Пункты вроде **публичная регистрация**, **платёжный шлюз (ЕРИП/карты)**, **Grafana/Prometheus**, **Kubernetes**, **managed PostgreSQL**, **S3 для документов**, **Celery для OCR**, **multi-tenant RLS**, **шардирование** — требуют продуктовых решений и/или облачной инфраструктуры. Детали остаются в [`../pilot/scaling-plan.md`](../pilot/scaling-plan.md); при планировании переносить сюда выборочно по приоритету.

## Продуктовые спринты 13–17 (дорожная карта)

Задачи **«production OCR / облачный движок»**, **ИИ-ассистент с RAG по регламентам**, **Push/Telegram**, **PWA/магазины приложений**, **нагрузочные k6/Locust на 1000+ клиентов**, **пентест/SOC2** — по умолчанию относятся сюда, пока не выбраны конкретные поставщики, бюджет и критерии готовности. Уточнения — в `scaling-plan`; реализуемая без контрактов часть выносится в [`TECH_CATCHUP_ORDERED.md`](TECH_CATCHUP_ORDERED.md).

## Спринт 18 (безопасность и комплаенс + перенос из 11–12)

OWASP-проход, шифрование PII «до стандарта», автоматизация backup, политика хранения, пентест — в основном **процессы и инфра**; отдельный перечень дублирующих отчётность задач см. таблицу выше.
