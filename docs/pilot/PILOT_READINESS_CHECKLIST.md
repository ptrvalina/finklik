# Pilot Readiness Checklist — команда внедрения

**1 страница · перед подключением каждого клиента**

Автоматическая часть: `make pilot-check` → `artifacts/pilot-readiness-scorecard.md`  
Ручная часть: этот чеклист (5–10 мин).

---

## A. Среда (до invite)

| # | Проверка | ☐ |
|---|----------|---|
| A1 | `make pilot-check` → **🟢** или **🟡** с понятными ограничениями | |
| A2 | PostgreSQL на prod (не SQLite) | |
| A3 | `JWT_SECRET_KEY` / refresh — не dev-дефолты | |
| A4 | `DEBUG=false` на API | |
| A5 | `CORS_ORIGINS` + `FRONTEND_URL` соответствуют SPA | |
| A6 | `EMAIL_API_KEY` или план ручной отправки invite | |
| A7 | Миграции: `alembic upgrade head` | |
| A8 | Клиенту отправлен [PILOT_SCOPE.md](./PILOT_SCOPE.md) (или PDF/summary) | |
