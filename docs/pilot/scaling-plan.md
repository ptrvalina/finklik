# План масштабирования ФинКлик
## 10 → 100 → 500 → 1000 клиентов

---

## Фаза 1: 10 → 100 клиентов (месяцы 1–3)

### Инфраструктура
- **Сервер:** 1 VPS (4 CPU, 8 GB RAM) — достаточно
- **БД:** PostgreSQL на том же сервере (до 50 кл.)
- **Redis:** одна нода 1 GB
- **Деплой:** Docker Compose
- **Стоимость:** ~50 USD/мес

### Что нужно сделать
- [ ] Открыть публичную регистрацию
- [ ] Настроить платёжный шлюз (ЕРИП / карты)
- [ ] Онбординг-автоматизация (welcome email, подсказки)
- [ ] Чат поддержки (Telegram бот)
- [ ] Мониторинг (Grafana + Prometheus)

### Метрики для перехода к Фазе 2
- 80+ активных платящих клиентов
- MRR > 4 600 BYN (~2 300 USD)
- Churn < 5%/мес

---

## Фаза 2: 100 → 500 клиентов (месяцы 3–9)

### Инфраструктура
- **БД:** Managed PostgreSQL (Yandex Cloud), 2 ноды
- **Redis:** Managed Redis, 2 GB
- **App:** 2–3 реплики API за балансировщиком
- **AI:** отдельный сервер для inference
- **CDN:** статика через Yandex CDN
- **Деплой:** Kubernetes (минимальный)
- **Стоимость:** ~300 USD/мес

### Что нужно сделать
- [ ] Перенос на Managed PostgreSQL
- [ ] Kubernetes деплой (через Helm)
- [ ] HPA: auto-scale API 2–10 реплик
- [ ] Разделить read/write трафик БД
- [ ] S3 для хранения документов
- [ ] Фоновые задачи (Celery/RQ для OCR)
- [ ] Партиционирование таблицы transactions по дате

### Метрики для перехода к Фазе 3
- 400+ активных клиентов
- MRR > 23 000 BYN (~11 500 USD)
- p95 latency < 300ms под нагрузкой
- Uptime > 99.5%

---

## Фаза 3: 500 → 1000 клиентов (месяцы 9–18)

### Инфраструктура
- **БД:** PostgreSQL cluster (master + 2 replica)
- **Cache:** Redis Cluster 3 ноды
- **App:** Kubernetes, 5–10 реплик, multi-zone
- **AI:** GPU inference (для ускорения OCR)
- **Queue:** RabbitMQ кластер
- **Стоимость:** ~1 000 USD/мес

### Что нужно сделать
- [ ] Multi-tenant изоляция данных (row-level security)
- [ ] Шардирование по org_id при >500 кл.
- [ ] Read replicas для аналитических запросов
- [ ] Message queue для всех async операций
- [ ] SOC 2 / ISO 27001 подготовка
- [ ] API rate limiting per-tenant
- [ ] Экспорт данных (GDPR-ready)
